# [BUG] review-prep generate URL without first pushing branch to remote

## Problem

The review-prep task generates a compare URL (e.g., `https://github.com/owner/repo/compare/dev...feature/NNN`) for developer review. This URL is always invalid because the feature branch has never been pushed to the remote. The branch is local-only — no `git push` occurs anywhere in the 14-step implementation pipeline.

## Root Cause

Two converging failures:

1. **review-prep.md assumes push already happened** (line 22: "Feature branch pushed (done by implementation task)"). But no implementation task in the pipeline pushes the branch. The closest step is `checkpoint-commit` (step 6) which dispatches `git-workflow --task commit-prep` — local commit only, no push.

2. **No push step in the 14-step pipeline.** The pipeline has `checkpoint-commit` (step 6) and `review-prep` (step 13), but neither includes `git push`. The branch is never sent to remote.

The result: every compare URL generated by review-prep is fake — the branch does not exist on GitHub, so the URL returns 404.

## Affected Files

- `.opencode/skills/git-workflow/tasks/review-prep.md` — line 16: "Branch MUST be pushed for developer review" is correct, but line 22: "Feature branch pushed (done by implementation task)" is false — no implementation task pushes
- `.opencode/skills/implementation-pipeline/tasks/pipeline-executor.md` — no push step in the 14-step dispatch routing table

## Solution Options

### Option A: Add push to review-prep (minimal)

Add a `git push -u origin HEAD` before generating the compare URL. Requires the remote to be configured (which it should be for any repo where a PR will be created).

### Option B: Add push to checkpoint-commit (structural)

Requires `git push origin <branch>` after every checkpoint commit in step 6. Ensures the branch always exists remotely throughout the pipeline.

### Option C: Extract push as its own step

Add `git-workflow --task push` as a new step between checkpoint-commit and structural-checks. Most invasive but cleanest separation of concerns.
