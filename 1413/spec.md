# [SPEC-FIX] Validate sub-agent skips pipeline-gate checks 11-16 — plans pass without mandatory implementation steps

## Problem

Two broken chains in the plan creation pipeline allow defective plans to pass validation and audit-fidelity.

### Chain 1: validate.md — all 16 checks are `(**sub-agent**)`, never execute

`writing-plans/tasks/validate.md` defines 16 validation checks (lines 9-87). Every single one is marked `(**sub-agent**)`. But the validate sub-agent is itself a sub-agent — it cannot dispatch sub-agents per the Mandatory Task Discipline rule 4 ("Sub-agents must not dispatch sub-agents"). So it reads a file full of steps labeled for someone else to do, and does nothing.

The formatting is correct per SC-SEM-006: checkboxes (`- [ ] N.`) for actionable steps, sub-bullets (`  - Command:`, `  - SC:`, `  - Expected:`) for parameter metadata. The only defect is the dispatch marker.

**Impact:** Checks 11 (pipeline-gate completeness) and 14 (phase workflow completeness) never run. Plans that skip mandatory implementation-pipeline gates pass validation.

### Chain 2: audit-fidelity.md → plan-fidelity.md — missing clean-room plan

`writing-plans/tasks/audit-fidelity.md` step 2 says "Execute `--task plan-fidelity` inline with auditor sub-agent type context." But `adversarial-audit/tasks/plan-fidelity.md` entry criteria require a `clean_room_plan` — plan text generated by a separate sub-agent dispatched via the writing-plans skill. Without it, `plan-fidelity` returns `BLOCKED` with `MISSING_CLEAN_ROOM_PLAN`.

The 21-step pipeline in `create.md` has no step between write (step 10) and audit-fidelity (step 16) that generates a clean-room plan. So `audit-fidelity` either BLOCKs or runs without a clean-room plan and checks nothing meaningful.

**Impact:** The fidelity audit is structurally broken — it cannot compare the plan against an independently-generated reference.

## Requirements

### R1: Change validate.md dispatch markers from `(**sub-agent**)` to `(**inline**)`

All 16 checks in `writing-plans/tasks/validate.md` must change from `(**sub-agent**)` to `(**inline**)`. The validate sub-agent runs these checks itself — it has all the tools it needs (grep, read, compare). No structural reformatting needed; the checkbox/sub-bullet structure is already correct.

### R2: Add clean-room plan generation step to the 21-step pipeline

Insert a new step between write (step 10) and revisit (step 12) in `writing-plans/tasks/create.md` that dispatches a clean-room sub-agent to generate a plan from the spec independently. The clean-room plan is passed as context to `audit-fidelity`, which passes it to `plan-fidelity` as `clean_room_plan`.

The new step:
- Step 11: (**sub-agent**) Clean-room plan generation — `task(..., prompt: "execute write task from writing-plans")` with spec body only, no existing plan context
- Chain: `step_10`
- Expected: clean_room_plan in output

### R3: Update audit-fidelity.md to pass clean_room_plan to plan-fidelity

`writing-plans/tasks/audit-fidelity.md` must accept `clean_room_plan` in its entry criteria and pass it as context when dispatching `plan-fidelity`.

### R4: Update write.md Plan Format Requirements to reference mandatory pipeline gates

`writing-plans/tasks/write.md` §Plan Format Requirements must explicitly state that every phase must include the full RED/GREEN chain with all mandatory implementation-pipeline gate steps from `implementation-pipeline/SKILL.md` §Dispatch Routing Table.
