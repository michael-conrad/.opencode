## Problem

Sub-agents are not capable of dispatching sub-agents. Despite this architectural invariant, multiple task cards across the skill deck contain `task()` calls that instruct the dispatched sub-agent to dispatch further sub-agents. This creates nested dispatch chains (orchestrator → sub-agent → sub-sub-agent) that cannot execute — sub-agents do not have access to the `task()` tool in the orchestrator's sense.

The root cause is a structural pattern error: task cards (which are consumed by sub-agents) contain orchestrator-level routing instructions. Only the main SKILL.md should contain the `[inline]`/`[sub-task]`/clean-room dispatch distinctions. Task cards describe procedures for a sub-agent to execute — they must not contain `task()` calls.

### Category Error: `analytical-artifacts.md`

`spec-creation-decomposition/tasks/analytical-artifacts.md` is a sub-agent task card whose content describes orchestrator-level behavior:

> "Each artifact is generated by the orchestrator dispatching a clean-room sub-agent via the SKILL.md Trigger Dispatch Table."
> "(*orchestrator*) Generate `blast-radius.yaml` — orchestrator dispatches via SKILL.md Trigger Dispatch Table."

A sub-agent reading this sees instructions about what the orchestrator should do — not actionable steps for the sub-agent itself. Sub-agents cannot call `skill()` or `task()` in the orchestrator's sense.

## Goals

- Establish the architectural invariant: **Only orchestrator-level SKILL.md files may contain `task()` dispatch calls. Task cards (consumed by sub-agents) MUST NOT contain `task()` calls.**
- Refactor all 9 violating task cards to remove `task()` calls and restructure dispatch logic into the appropriate SKILL.md Trigger Dispatch Tables.
- Fix the `analytical-artifacts.md` category error: convert from orchestrator-level instructions to sub-agent-executable procedure, or move orchestrator-level content to the SKILL.md.

## Non-Goals

- Not changing the overall pipeline architecture — only fixing the dispatch boundary violation
- Not modifying skills outside spec-creation and writing-plans (though the same pattern may exist elsewhere)

## Approach

For each violating task card, the fix pattern is:
1. Move the `task()` dispatch call from the task card to the parent SKILL.md Trigger Dispatch Table
2. The task card becomes a pure sub-agent procedure (no `task()` calls, no orchestrator-level instructions)
3. The SKILL.md gains new dispatch entries for the steps that were previously dispatched from within task cards
