# Planning: Status Tracking — REMOVED

**This guideline has been removed.** Specs and plans are NOT tracking documents. They define what is required — implemented or not. Implementation progress is tracked through pipeline state (work state files, lifecycle manifests, PR status), never through tracking fields in the spec or plan body.

See `/AGENTS.md` §Specs and Plans Are NOT Tracking Documents and `.opencode/AGENTS.md` §Specs and Plans Are NOT Tracking Documents for the authoritative rule.
